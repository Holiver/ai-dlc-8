## 系统简介

### 什么是 AWSomeShop？

AWSomeShop 是一个员工福利电商系统，允许员工使用积分兑换公司提供的福利产品。系统支持中英双语，可在电脑和手机上使用。

### 核心功能

**员工功能：**
- 浏览和兑换福利产品
- 查看积分余额和历史
- 查看兑换历史
- 管理个人信息

**管理员功能：**
- 管理员工账户
- 管理产品和库存
- 发放和扣除积分
- 管理订单状态
- 查看统计报表

### 系统要求

- 现代浏览器（Chrome、Firefox、Safari、Edge）
- 稳定的网络连接
- 推荐分辨率：1366x768 或更高

## 员工端使用指南

### 1. 登录系统

#### 首次登录

1. 打开浏览器，访问系统地址
2. 在登录页面输入：
   - **邮箱**：公司分配的邮箱地址
   - **密码**：管理员提供的初始密码
3. 点击"登录"按钮
4. 首次登录成功后，系统会自动发放 **1000 积分**

![登录页面示例](docs/images/login.png)

#### 后续登录

使用相同的邮箱和密码登录即可。

#### 忘记密码？

请联系管理员重置密码。

---

### 2. 浏览产品

登录后，系统会自动跳转到产品列表页面。

#### 产品信息

每个产品卡片显示：
- **产品图片**：产品的展示图片
- **产品名称**：产品的名称
- **产品描述**：产品的详细说明
- **所需积分**：兑换该产品需要的积分数量
- **库存状态**：
  - 绿色"有货"：可以兑换
  - 红色"缺货"：暂时无法兑换

#### 查看积分余额

在页面顶部可以看到：
- **当前积分余额**：您可用的积分总数
- 积分余额会实时更新

![产品列表页面](docs/images/products.png)

---

### 3. 兑换产品

#### 兑换步骤

1. 在产品列表中找到想要的产品
2. 确认：
   - 积分余额足够
   - 产品有库存
3. 点击产品卡片上的"兑换"按钮
4. 在弹出的确认对话框中：
   - 查看产品信息
   - 查看所需积分
   - 确认兑换
5. 点击"确认"按钮
6. 等待系统处理
7. 看到成功提示后，兑换完成

#### 兑换后

- 积分余额会立即减少
- 订单会出现在"兑换历史"中
- 积分变动会记录在"积分历史"中

#### 注意事项

- 兑换前请仔细确认产品信息
- 兑换成功后无法取消
- 积分扣除后无法退回
- 产品发放由管理员处理

![兑换确认对话框](docs/images/redemption-confirm.png)

---

### 4. 查看兑换历史

#### 访问兑换历史

1. 点击顶部导航栏的"兑换历史"
2. 或点击侧边栏的"兑换历史"菜单

#### 订单信息

每条订单记录显示：
- **订单号**：唯一的订单编号
- **产品名称**：兑换的产品
- **积分数量**：使用的积分
- **订单状态**：
  - 🟡 **待处理**：订单已创建，等待发放
  - 🟢 **已发放**：产品已发放，可以领取
  - 🔴 **已取消**：订单已取消
- **创建时间**：兑换的时间

#### 订单排序

订单按创建时间倒序排列，最新的订单在最上面。

#### 分页浏览

如果订单较多，可以使用底部的分页控件：
- 点击页码跳转
- 选择每页显示数量（10/20/50）

![兑换历史页面](docs/images/redemption-history.png)

---

### 5. 查看积分历史

#### 访问积分历史

1. 点击顶部导航栏的"积分历史"
2. 或点击侧边栏的"积分历史"菜单

#### 交易记录

每条积分记录显示：
- **交易类型**：
  - 🎁 **首次登录奖励**：首次登录获得
  - ➕ **积分发放**：管理员发放
  - ➖ **积分扣除**：管理员扣除
  - 🛒 **产品兑换**：兑换产品消费
- **积分数量**：
  - 正数表示增加
  - 负数表示减少
- **备注**：交易的说明信息
- **交易时间**：发生的时间

#### 余额计算

当前积分余额 = 所有交易的总和

#### 分页浏览

使用底部的分页控件浏览历史记录。

![积分历史页面](docs/images/points-history.png)

### 6. 管理个人信息

#### 访问个人信息

1. 点击顶部导航栏的"个人信息"
2. 或点击侧边栏的"个人信息"菜单

#### 查看信息

可以查看：
- **姓名**：您的全名
- **邮箱**：登录邮箱（不可修改）
- **手机号**：联系电话
- **积分余额**：当前可用积分
- **账户状态**：激活/离职

#### 修改手机号

1. 在"手机号"字段输入新号码
2. 点击"保存"按钮
3. 等待成功提示
4. 刷新页面验证修改

#### 注意事项

- 邮箱地址不能修改
- 如需修改其他信息，请联系管理员

![个人信息页面](docs/images/profile.png)

---

### 7. 语言切换

#### 切换语言

1. 点击页面右上角的语言切换按钮
2. 选择：
   - 🇨🇳 **中文**
   - 🇺🇸 **English**
3. 页面会立即切换到选择的语言

#### 语言设置

- 语言设置会保存在浏览器中
- 下次登录会自动使用上次选择的语言

### 8. 退出登录

#### 退出步骤

1. 点击页面右上角的用户名
2. 在下拉菜单中点击"退出登录"
3. 系统会清除登录状态
4. 自动跳转到登录页面

#### 安全建议

- 使用公共电脑后请务必退出登录
- 不要与他人分享您的密码

## 管理员端使用指南

### 1. 管理员登录

使用管理员账户登录后，会自动跳转到管理员仪表板。

![管理员仪表板](docs/images/admin-dashboard.png)

### 2. 员工管理

#### 访问员工管理

点击侧边栏的"员工管理"菜单。

#### 查看员工列表

员工列表显示：
- 姓名
- 邮箱
- 手机号
- 积分余额
- 账户状态（激活/离职）
- 创建时间

#### 创建员工账户

1. 点击"创建员工"按钮
2. 填写员工信息：
   - **姓名**：员工的全名
   - **邮箱**：用于登录的邮箱（必须唯一）
   - **手机号**：联系电话
3. 点击"创建"按钮
4. 系统会自动生成初始密码
5. **重要**：记录并告知员工初始密码

#### 初始密码规则

系统自动生成的初始密码格式：
- 8 位随机字符
- 包含大小写字母和数字
- 例如：`Abc12345`

#### 设置员工离职

1. 在员工列表中找到目标员工
2. 点击"设置离职"按钮
3. 确认操作
4. 员工账户会被标记为"离职"
5. 该员工将无法登录系统
6. 积分余额会被清零

#### 注意事项

- 邮箱地址必须唯一
- 离职操作不可撤销
- 离职员工的历史记录会保留

![员工管理页面](docs/images/admin-users.png)

---

### 3. 产品管理

#### 访问产品管理

点击侧边栏的"产品管理"菜单。

#### 查看产品列表

产品列表显示：
- 产品名称
- 描述
- 所需积分
- 库存数量
- 状态（已上架/已下架）
- 创建时间

#### 创建产品

1. 点击"创建产品"按钮
2. 填写产品信息：
   - **产品名称**：产品的名称
   - **描述**：产品的详细说明
   - **图片 URL**：产品图片的网址
   - **所需积分**：兑换需要的积分数量
   - **库存**：可兑换的数量
3. 点击"创建"按钮
4. 产品默认为"已上架"状态

#### 编辑产品

1. 在产品列表中找到目标产品
2. 点击"编辑"按钮
3. 修改产品信息
4. 点击"保存"按钮
5. 如果修改了积分，系统会自动记录价格历史

#### 上架/下架产品

**下架产品：**
1. 点击产品的"下架"按钮
2. 确认操作
3. 产品状态变为"已下架"
4. 员工端将看不到该产品

**上架产品：**
1. 点击产品的"上架"按钮
2. 确认操作
3. 产品状态变为"已上架"
4. 员工端可以看到该产品

#### 批量导入产品

1. 点击"批量导入"按钮
2. 在文本框中输入 Markdown 表格格式的产品数据：

```markdown
| 产品名称 | 描述 | 图片URL | 所需积分 | 库存 |
|---------|------|---------|---------|------|
| 无线鼠标 | 罗技无线鼠标 | https://example.com/mouse.jpg | 100 | 50 |
| 机械键盘 | 樱桃轴机械键盘 | https://example.com/keyboard.jpg | 300 | 30 |
| 蓝牙耳机 | 索尼降噪耳机 | https://example.com/headphone.jpg | 500 | 20 |
```

3. 点击"导入"按钮
4. 系统会解析表格并创建所有产品
5. 查看导入结果

#### 注意事项

- 产品名称建议简洁明了
- 图片 URL 必须是有效的网址
- 积分和库存必须是正整数
- 下架的产品不会删除，只是隐藏

![产品管理页面](docs/images/admin-products.png)

### 4. 积分管理

#### 访问积分管理

点击侧边栏的"积分管理"菜单。

#### 发放积分（单个员工）

1. 选择"发放积分"标签
2. 填写发放信息：
   - **员工邮箱**：接收积分的员工邮箱
   - **积分数量**：发放的积分数量（正整数）
   - **备注**：发放原因或说明
3. 点击"发放"按钮
4. 等待成功提示
5. 员工的积分余额会立即增加

#### 批量发放积分

1. 选择"批量发放"标签
2. 在文本框中输入 Markdown 表格格式的数据：

```markdown
| 员工邮箱 | 积分数量 | 备注 |
|---------|---------|------|
| employee1@example.com | 100 | 月度绩效奖励 |
| employee2@example.com | 200 | 季度优秀员工 |
| employee3@example.com | 150 | 项目完成奖励 |
```

3. 点击"批量发放"按钮
4. 系统会解析表格并发放积分
5. 查看发放结果

#### 扣除积分

1. 选择"扣除积分"标签
2. 填写扣除信息：
   - **员工邮箱**：被扣除积分的员工邮箱
   - **积分数量**：扣除的积分数量（正整数）
   - **备注**：扣除原因或说明
3. 点击"扣除"按钮
4. 系统会验证员工积分余额
5. 如果余额足够，扣除成功

#### 注意事项

- 发放数量必须是正整数
- 扣除数量不能大于员工余额
- 批量操作中，如果某个员工邮箱无效，该条记录会失败，其他记录继续处理
- 所有操作都会记录在积分历史中

![积分管理页面](docs/images/admin-points.png)

### 5. 订单管理

#### 访问订单管理

点击侧边栏的"订单管理"菜单。

#### 查看订单列表

订单列表显示：
- 订单号
- 员工姓名和邮箱
- 产品名称
- 积分数量
- 订单状态
- 创建时间

#### 筛选订单

使用状态筛选器：
- **全部**：显示所有订单
- **待处理**：显示待处理的订单
- **已发放**：显示已发放的订单
- **已取消**：显示已取消的订单

#### 更新订单状态（单个）

1. 在订单列表中找到目标订单
2. 点击"更新状态"按钮
3. 选择新状态：
   - **待处理**：订单已创建，等待处理
   - **已发放**：产品已发放给员工
   - **已取消**：订单已取消
4. 确认操作
5. 订单状态更新

#### 批量更新订单状态

1. 勾选要更新的订单（可多选）
2. 点击"批量更新状态"按钮
3. 选择新状态
4. 确认操作
5. 所有选中的订单状态更新

#### 使用场景

**待处理 → 已发放：**
- 产品已准备好并发放给员工
- 员工可以领取产品

**待处理 → 已取消：**
- 产品缺货无法发放
- 员工申请取消
- 其他特殊情况

#### 注意事项

- 订单状态更新后无法撤销
- 取消订单不会退还积分（需要管理员手动发放）
- 建议定期处理待处理订单

## 常见问题

### 员工端问题

#### Q1: 忘记密码怎么办？

**A:** 请联系管理员重置密码。管理员可以为您生成新的初始密码。

#### Q2: 首次登录没有获得积分？

**A:** 请检查：
1. 是否真的是首次登录
2. 联系管理员查看系统日志
3. 查看积分历史是否有"首次登录奖励"记录

#### Q3: 兑换时提示积分不足？

**A:** 您的积分余额不足以兑换该产品。请：
1. 查看积分余额
2. 选择所需积分更少的产品
3. 等待管理员发放更多积分

#### Q4: 兑换时提示库存不足？

**A:** 该产品暂时缺货。请：
1. 选择其他产品
2. 等待管理员补充库存
3. 联系管理员了解补货时间

#### Q5: 兑换成功后如何领取产品？

**A:** 
1. 在"兑换历史"中查看订单状态
2. 等待状态变为"已发放"
3. 按照公司规定的流程领取产品
4. 具体领取方式请咨询管理员

#### Q6: 可以取消已兑换的订单吗？

**A:** 兑换成功后无法自行取消。如有特殊情况，请联系管理员处理。

#### Q7: 积分会过期吗？

**A:** 
- 正常情况下积分不会过期
- 如果员工离职，积分会被清零
- 具体规则请咨询管理员

#### Q8: 为什么看不到某个产品？

**A:** 可能的原因：
1. 产品已下架
2. 产品库存为 0
3. 产品已被删除

#### Q9: 手机号修改后为什么没有生效？

**A:** 请：
1. 确认点击了"保存"按钮
2. 刷新页面查看
3. 如果仍未生效，联系管理员

#### Q10: 可以在手机上使用吗？

**A:** 可以。系统支持响应式设计，可以在手机浏览器中正常使用。

### 管理员端问题

#### Q1: 如何重置员工密码？

**A:** 目前系统不支持密码重置功能。建议：
1. 将员工设置为离职
2. 重新创建员工账户
3. 手动发放原有积分

#### Q2: 批量导入失败怎么办？

**A:** 请检查：
1. Markdown 表格格式是否正确
2. 表头是否完全匹配
3. 数据类型是否正确（积分和库存必须是数字）
4. 是否有空行或多余字符

#### Q3: 如何查看产品的价格历史？

**A:** 目前系统会自动记录价格变更，但没有提供查询界面。如需查看，请联系技术支持查询数据库。

#### Q4: 员工离职后可以恢复吗？

**A:** 离职操作不可撤销。如需恢复，只能：
1. 重新创建员工账户
2. 手动发放积分

#### Q5: 如何处理积分发放错误？

**A:** 
1. 如果多发了积分，使用"扣除积分"功能扣回
2. 如果少发了积分，使用"发放积分"功能补发
3. 所有操作都会记录在积分历史中

#### Q6: 订单取消后积分会退还吗？

**A:** 不会自动退还。如需退还，请：
1. 将订单状态更新为"已取消"
2. 手动使用"发放积分"功能退还积分

#### Q7: 如何防止库存超卖？

**A:** 系统已实现并发控制，使用数据库事务确保库存准确性。多个用户同时兑换时，系统会自动处理。

#### Q8: 报表导出的 CSV 文件乱码怎么办？

**A:** 
1. 使用 Excel 打开时，选择"数据" → "从文本/CSV"
2. 选择编码为 UTF-8
3. 或使用 Google Sheets 打开（自动识别编码）

#### Q9: 如何备份数据？

**A:** 请参考《部署指南》中的"数据备份策略"章节。

#### Q10: 系统支持多少并发用户？

**A:** 系统设计支持 100 并发用户。如需支持更多用户，请联系技术支持进行性能优化。
